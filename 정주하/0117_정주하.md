## 네이티브 WebSocket

- 단순한 양방향 메시지 송수신
- 가볍고 성능에 최적화
- 추가 종속성이 없으며 별도의 모듈 설치 불필요
- 멀티 플렉싱이 내장되지 않아 브로드캐스팅이 불가
- fallback 옵션이 없어 연결되지 않으면 자동 복구를 시도하지 않음
- Stateless 프로토콜이기 때문에 클라이언트의 이전 요청과 상태 정보에 대한 의존성이 없다.

### 1. Opening Handshake

```
GET ws://localhost:3000/ HTTP/1.1 
Host: localhost: 3000
Connection: Upgrade
Upgrade: websocket
Sec-WebSocket-Version: 13
Sec-WebSocket-Key: zy6Dy9mSAIM7GJZNf9rI1A==
```

**클라이언트→ 서버**

- HTTP를 WebSocket 프로토콜로 업그레이트 해달라는  GET 요청
- Sec-WebSocket-Version은 13만 허용
- Sec-WebSocket-Key는 클라이언트가 서버에 보내는 일종의 key이며 다른 요청에 서버가 응답해선 안 됨

```
HTTP/1.1 101 Switching Protocols
Connection: Upgrade
Sec-WebSocket-Accept: EDJa7WCAQQzMCYNJM42Syuo9SqQ=
Upgrade: websocket
```

**서버→ 클라이언트**

- Sec-WebSocket-Accept로 클라이언트의 key에 대한 응답을 보냄

API를 사용하면 프로토콜은 알아서 처리되기에 new 생성자 함수로 소켓을 만들어 open해준다.

```jsx
// 소켓 연결 후 데이터 전달
const socket = new Websocket('ws://localhost:3000');

socket.onopen = (e) => {
	console.log('소켓 연결');
};
```

암호화된 주소를 사용한다면 wss를 사용

### 2. 데이터 주고받기

```jsx
socket.onmessage = (e) => {
  	console.log(JSON.parse(e));//서버 메시
	socket.send(JSON.stringify({'userId': 'hello1234'}));
};
```

- 웹소켓은 비동기 방식의 이벤트 중심 프로그래밍
- 데이터를 주고 받거나 연결 상태 변경할 때 이벤트를 활용
- 메세지는 string이나 binary 형식이어야 하기 때문에 직렬화 및 역직렬화가 가능한 JSON 형태로 보내야한다.

### 3.연결 종료

```jsx
socket.onclose = (e) => {
	console.log('웹소켓 연결 종료');
};
```

- Socket이 오픈되어 있다면 연결은 끊기 전까지 지속되므로 더 이상 통신이 필요 없는 경우 연결을 종료

### 에러 처리

```
socket.onerror = (e) => {
    console.error('WebSocket 오류:', e);
};
```

### WebSocket을 연결하고 싶다면? : socket.iO

## Socket.IO

- **내장된 멀티플렉싱으로 브로드캐스팅이 가능하다.**
    - 서버에서 여러 클라이언트에게 동시에 메시지를 보낼 때 각 연결을 개별적으로 하지 않고 단 하나의 연결로 여러 작업을 처리한다.
- **네임스페이스와 룸**
    - 네임스페이스는 소켓 통신을 분리하는 고유 식별자
    - 룸은 네임스페이스 내에서 클라이언트를 **그룹화**하고 관리
    - 여러 클라이언트를 하나로 그룹화하여 특정 사용자들에게만 메시지를 전송할 때 사용할 수 있다.
    - 하나의 네임스페이스는 여러 룸을 가질 수 있다.
- **fallback 옵션이 있어 자동 재연결 시도 가능하며 대기 시간과 시도 횟수를 설정 가능하다.**
- **HTTP Polling으로 먼저 소켓 연결 설정**
    - 이후 WebSocket으로 업그레이드
    - 업그레이드 되지 않더라도 **HTTP 폴링**으로 연결을 미리 시도하기 때문에 연결 오류에 대응할 여력이 줄어듬
- **메시지 핸들링**
    - 이벤트를 사용자 정의 가능
- **다중 플랫폼 지원**
    - node.js가 아닌 java 또는 python과 같은 다른 백엔트 언어 사용 가능
- **상태 저장 및 관리 기능**
    - 클라이언트의 상태 정보를 저장하며 상태를 공유할 수도 있다.
    - 클라이언트의 세션 상태, 사용자 인증 정보의 활용이 가능


    **1. 서버 설정**

Node.js 서버에 [Socket.IO](http://Socket.IO) 설치

```bash
npm install socket.io
```

서버 코드

```jsx
<--기본 설정-->
const express = require('express');
const http = require('http');
const socketIo = require('socket.io');

const app = express();
const server = http.createServer(app);
const io = socketIo(server);
```

```jsx
//소켓연결과 메시지 수신 및 전달
io.on('connection', (socket) => {
		io.emit("message", `${socket.id} 님이 연결되었습니다.`);
    console.log('소켓 연결');

    socket.on('chat message', (msg) => {
        console.log('메시지: ' + msg);
        // 모든 클라이언트에게 메시지 전송
        io.emit('chat message', msg);
    });
    
    //연결이 끊기면 클라이언트에게 알리기
    socket.on("disconnect", () => {
	    socket.broadcast.emit("message", `${socket.id} 님의 연결이 끊어졌습니다.`);
	  });
  
});

server.listen(3000, () => {
    console.log('3000 Port Open');
});

```

1. **클라이언트 설정**

Node.js 환경에서 설치

```bash
npm install socket.io-client
```

브라우저에선 CDN으로도 사용 가능

```html
<script src="https://cdn.socket.io/4.5.4/socket.io.min.js"></script>
```

클라이언트 코드

```jsx
const io = require('socket.io-client');
const socket = io('http://localhost:3000');

// 서버 연결 성공 시
socket.on('connect', () => {
    console.log('Socket.IO 연결 성공:', socket.id);

    // 서버에 메시지 전송
    socket.emit('message', { userId: 'hello1234' });
});

// 서버에서 메시지 수신
socket.on('welcome', (data) => {
    console.log('서버 메시지:', data);
});

// 서버 응답 처리
socket.on('response', (data) => {
    console.log('서버 응답:', data);
});

// 연결 종료 시
socket.on('disconnect', (reason) => {
    console.log('Socket.IO 연결 종료:', reason);
});

```

### **Socket.IO vs WebSocket 비교**
| **특징** | **Socket.IO** | **WebSocket** |
| --- | --- | --- |
| **기본 프로토콜** | WebSocket + 폴백 (롱 폴링 등) | WebSocket |
| **자동 재연결** | 기본 제공 | 직접 구현 필요 |
| **이벤트 기반 메시징** | 지원 (네임스페이스와 룸 포함) | 직접 구현 필요 |
| **브라우저 호환성** | 더 좋음 (폴백 지원) | WebSocket을 지원하는 브라우저에서만 사용 가능 |
| **확장성** | 네임스페이스, 룸, 클러스터링 등 추가 기능 제공 | 추가 기능은 직접 구현해야 함 |
| **성능** | 약간 무거움 (추가 기능과 폴백 처리 때문) | 더 가벼움 |
| **사용 예** | 대규모 실시간 서비스, 이벤트 기반 애플리케이션 | 단순한 양방향 통신 |
