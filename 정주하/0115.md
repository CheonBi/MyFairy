# Transmission control Protocol/Internet Protocol

---

## TCP와 IP란?

### IP

- **라우팅**과 **주소지정**
- 비신뢰성
- 비연결성

### TCP

- 전송 제어 프로토콜
- **3 Way-Handshake**
    1. **클라이언트**: 상대에게 통신 요청 메세지 전송 (SYN) 
    2. **서버**: 메세지에 대한 응답과 통신 준비 완료 메세지 전송 (SYN-ACK)
    3. **클라이언트**: 2번에서 받은 메세지에 응답을 보낸다 (ACK)
    
    → 신뢰성 보장, 양방향 확인. 순차적 통신 보장
    
- 패킷의 손실 없이 안전하게 전달→ **데이터 전송의 신뢰성을 보장**
- 혼잡 제어 및 흐름제어
    - 데이터 손실이나 과부하 방지

*즉, TCP/IP 통신은  IP의 문제를 보완하며 안정적이고 효율적인 데이터 전송을 보장한다.*

## TCP/IP 특징

- **연결 지향적 (TCP)**
    - TCP는 연결 지향적 프로토콜이다
    - 신뢰성을 보장하기 위해 데이터가 손실되거나 순서가 변겯외면 다시 전송한다
- **패킷 전송**
    - 데이터를 작은 패킷 단위로 나누어 전송→ IP+순서번호 포함
    - 여러 패킷을 재조합하여 원본 데이터 복원
- **패킷 손실 및 오류 처리**
    - TCP는 재전송 기능을 통해 데이터가 손실되거나 오류가 발생할 경우 자동으로 다시 전송하여 신뢰성을 보장
- **흐름 제어 및 혼잡 제어**
    - 네트워크의 상태에 맞게 데이터 전송 속도를 조절하여 네트워크의 성능 저하를 방지하고, 트래픽 혼잡을 해결한다.

## TCP/IP 4Layer


### **Application Layer**

특정 서비스를 제공하기 위해 어플리케이션끼리의 정보를 주고 받는 층이다. 

→ SMTP, HTTP, FTP, SSH, Talnet, DNS 등 다양한 프로토콜이 있다

### **Transport Layer**

송신된 데이터를 수신측 어플리케이션에 확실하게 전달하기 위한 층

어플리케이션 각각의 PORT 번호를 통해 어떤 어플리케이션에 전달할 것인지 찾아주는 역할

→ TCP, UDP, RTP, RCTP등의 프로토콜

### **Internet Layer**

수신측까지 데이터를 전달하기 위해 사용되는 층

어떤 네트워크 장비에 라우팅할것인지 결정하는 층

→ IP, ARP, ICMP, RARP, OSPF등의 프로토콜이 있다

### **Network Access Layer**

네트워크에 연결된 물리적인 기기 간 전송을 담당하는 층

MAC 주소가 이 계층에서 사용된다→ 실제 디지털 신호를 주고받는 층

→ Ethernet, TTokenRing, FDDI등의 프로토콜이 있다.
