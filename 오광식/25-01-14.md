# JWT(JSON Web Token) Storage and Management

JWT(JSON Web Token)를 저장하고 관리하는 방식은 보안과 사용성 측면에서 중요합니다. 
일반적으로 클라이언트에서 JWT를 저장하는 방식은 아래 두 가지

---

### 1. **로컬 스토리지(Local Storage)**
   - **특징**:
     - 클라이언트 측에서 데이터를 영구적으로 저장할 수 있는 공간.
     - 브라우저가 종료되어도 데이터가 유지됩니다.
   - **장점**:
     - 구현이 간단합니다.
     - 브라우저의 개발자 도구로 쉽게 확인 및 디버깅 가능.
   - **단점**:
     - XSS(Cross-Site Scripting) 공격에 취약하여 토큰이 탈취될 위험이 있습니다.

   ```javascript
   // JWT 저장
   localStorage.setItem("jwtToken", token);

   // JWT 가져오기
   const token = localStorage.getItem("jwtToken");
   ```

---

### 2. **세션 스토리지(Session Storage)**
   - **특징**:
     - 세션 기반 저장소로, 브라우저 탭이 닫히면 데이터가 삭제됩니다.
   - **장점**:
     - XSS 공격에 대한 위험이 로컬 스토리지보다 낮습니다(세션 범위로 제한됨).
   - **단점**:
     - 브라우저가 닫히면 로그인이 유지되지 않습니다.
   - **사용 예**:
     ```javascript
     // JWT 저장
     sessionStorage.setItem("jwtToken", token);

     // JWT 가져오기
     const token = sessionStorage.getItem("jwtToken");
     ```

---

### 3. **쿠키(Cookies)**
   - **특징**:
     - 서버와 클라이언트 간 자동으로 전달되는 HTTP 요청의 일부로 사용.
     - `HttpOnly` 속성을 사용하면 JavaScript로 접근할 수 없으므로 XSS 공격 방지에 효과적입니다.
   - **장점**:
     - CSRF(Cross-Site Request Forgery) 공격을 방지할 수 있습니다(`SameSite` 속성 설정).
     - 서버-클라이언트 간 자동 전송.
   - **단점**:
     - 설정 및 관리가 상대적으로 복잡합니다.
   - **설정 예**:
     ```java
     // Spring Boot - JWT 쿠키 설정
     ResponseCookie jwtCookie = ResponseCookie.from("jwtToken", token)
         .httpOnly(true)
         .secure(true)
         .sameSite("Strict")
         .path("/")
         .build();
     response.addHeader(HttpHeaders.SET_COOKIE, jwtCookie.toString());
     ```

---

### **비교 및 선택 가이드**
| 저장 방식         | 보안성  | 유지성 | 구현 난이도 | 추천 상황              |
|--------------------|---------|--------|--------------|------------------------|
| 로컬 스토리지     | 낮음    | 영구적 | 쉬움         | 단순 SPA 애플리케이션. |
| 세션 스토리지     | 중간    | 세션   | 쉬움         | 브라우저 세션 동안만 인증 필요. |
| 쿠키              | 높음    | 영구적 | 보통         | 민감한 데이터 보호 필요. |

---

### **베스트 프랙티스**
1. **짧은 수명**: JWT의 만료 시간을 짧게 설정하여 보안성을 높입니다.
2. **갱신 토큰 사용**: 액세스 토큰(JWT)과 리프레시 토큰을 조합하여 사용합니다.
3. **Https**: 반드시 HTTPS 프로토콜을 사용하여 네트워크 보안성을 강화합니다.
4. **HttpOnly** 및 **Secure** 쿠키: 가능하면 쿠키를 통해 관리하고, 관련 옵션을 사용하여 보안성을 높입니다.

어떤 저장 방식을 사용할지 결정할 때는 애플리케이션의 요구 사항과 보안 수준을 고려
